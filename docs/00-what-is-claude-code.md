# 第 0 章：Claude Code 是什么？

> **本章目标**：用大白话建立直觉，不需要任何编程背景。

---

## 先用生活场景理解

想象你雇了一个助理，你告诉他：「帮我整理一下这个月的财务报表，把所有超过 1000 元的支出单独列出来，算出总额，然后发给我。」

一个**普通的 AI 工具**会这样做：告诉你「你应该这样整理……」，给你一堆建议，但什么都不帮你做。

一个**更好的 AI 工具**会这样做：帮你写一段代码，但你还得自己去运行它，看结果，发现有问题再回来问它。

**Claude Code** 会这样做：直接打开你的文件夹，读取财务数据，自己写代码处理，自己运行，发现数字对不上就自己调试，直到结果正确，最后把整理好的报表放到你指定的位置，告诉你「完成了」。

这就是 Claude Code 和其他 AI 工具的本质区别：**它不只是给建议，它真的动手做事。**

---

## AI 工具的三个进化阶段

理解 Claude Code 的位置，需要先看 AI 编程工具的进化历程：

| 阶段 | 代表工具 | 能做什么 | 局限 |
|------|---------|---------|------|
| 第一代：代码补全 | GitHub Copilot | 根据上下文补全代码 | 只能在你写代码时提示，不能主动行动 |
| 第二代：对话助手 | ChatGPT、IDE 聊天 | 回答问题、生成代码片段 | 生成的代码需要你手动复制、运行、验证 |
| 第三代：自主 Agent | **Claude Code** | 自主执行多步操作，直到任务完成 | 需要在你的电脑上运行，有安全风险需要管控 |

Claude Code 属于第三代——**自主 Agent**。它不只是生成文本，而是能真正在你的电脑上执行操作。

---

## 一个真实的例子

你告诉 Claude Code：「给这个项目添加用户登录功能」。

它会自主完成以下所有步骤，不需要你一步步指导：

1. 用 `grep` 搜索现有代码，理解项目用的是什么框架
2. 读取数据库配置文件，了解数据库结构
3. 创建用户表的数据库迁移文件
4. 编写登录 API 接口
5. 编写前端登录页面
6. 运行测试，发现有一个测试失败
7. 读取错误信息，定位问题
8. 修复代码，再次运行测试
9. 测试通过，提交 git commit

整个过程是一个**自主决策循环**：AI 决定下一步做什么，执行后观察结果，再决定下一步。你只需要在最后审核结果，或者在某些危险操作时确认一下。

---

## 它是怎么做到的？

Claude Code 能做到这一切，依靠三个核心能力：

**工具（Tools）**：它有 66 个内置工具，包括读文件、写文件、执行命令、搜索代码、调用 API 等。每次需要「做点什么」，它就调用对应的工具。

**循环（Loop）**：它不是「问一次答一次」，而是在一个循环里不断「思考 → 行动 → 观察结果 → 再思考」，直到认为任务完成。

**记忆（Memory）**：它能记住你的项目偏好、团队约定、历史教训，下次开新对话时不需要重新解释。

## Claude Code vs 其他 AI 工具

很多人会问：这和 Cursor、GitHub Copilot、Devin 有什么区别？

| 工具 | 定位 | 主要场景 | 局限 |
|------|------|---------|------|
| GitHub Copilot | 代码补全 | 在你写代码时提示 | 被动响应，不主动执行 |
| Cursor | AI 增强的 IDE | 在 IDE 里用 AI 改代码 | 需要人手动确认每个改动 |
| Devin | 自主编程 Agent | 完全自主的开发任务 | 云端运行，较慢，较贵 |
| **Claude Code** | **终端自主 Agent** | **在你的电脑上直接操作** | 需要信任和安全意识 |

Claude Code 的核心差异在于：**它直接在你的本地环境中运行**，可以访问你的文件系统、运行你的命令、使用你的工具。这带来了极大的灵活性，也带来了需要谨慎对待的安全问题。

## 源码泄露事件

这份教程基于一件真实事件：2025 年初，Claude Code 的完整源码（512,000+ 行 TypeScript）被意外泄露到了 npm 包中。

注意：**Anthropic 从未公开过 Claude Code 的源码**。这件事属于意外泄露，不是官方公开。泄露的内容包括：

- 完整的 TypeScript 源码（虽然经过混淤）
- 内部系统提示词
- 功能标志配置
- 内部工具定义

这为我们提供了一个难得的机会：不靠猜测，而是直接阅读一个世界一流的 AI 编程工具的内部实现。
